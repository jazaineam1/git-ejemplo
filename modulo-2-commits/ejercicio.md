# 🧪 Ejercicio 2: Escribir un commit claro

## Contexto

En este ejercicio practicarás algo sencillo técnicamente, pero muy importante profesionalmente: describir bien un cambio.

## Tarea

1. Crear archivo `mejoras.txt`
2. Escribir 3 mejoras imaginarias para un sistema
3. Hacer un commit con mensaje claro

## Ejemplo de mensaje esperado

```text
Agrega lista de mejoras del sistema
```

## Qué se evalúa

- que el archivo exista
- que el contenido tenga sentido
- que el commit explique el cambio

## Reflexión

Si otra persona viera tu commit sin abrir el archivo, ¿entendería qué agregaste?
