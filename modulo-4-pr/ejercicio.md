# 🧪 Ejercicio 4: PR completo

## Contexto

Este ejercicio simula una entrega real de trabajo. No basta con cambiar un archivo: debes completar todo el flujo profesional.

## Tarea

1. Crear rama:

```bash
git switch -c feature/readme
```

2. Editar `README.md`
3. Hacer commit con mensaje claro
4. Publicar la rama:

```bash
git push origin feature/readme
```

5. Ir a GitHub
6. Crear Pull Request

## ✍️ Descripción obligatoria del PR

Tu PR debe incluir al menos:

- qué hiciste
- por qué lo hiciste
- cómo revisar el cambio

## 🎯 Evaluación

- rama correcta
- commit claro
- PR bien escrito
- cambio coherente con la tarea

## Consejo

Si el cambio fuera visual o de interfaz, en un entorno real conviene adjuntar evidencia como capturas antes y después.
